Thesis

代码层第一次开始形成高层闭环

过去能闭环的是语法、测试和 lint。现在，代码层开始在“理解问题、修改实现、接收反馈”这一层形成更高阶的回路。

Watt Governor：传感器是转速，执行器是蒸汽阀，工人从“手动拧阀门”转向“设计调速器与阈值”。
Kubernetes：工程师声明 desired state，controller 持续观察 actual state，并自动 reconcile 到目标状态。
共同模式：有了足够好的传感器和执行器，人就会从“亲自做动作”转向“设计目标与反馈”。
01 / Control Loop Lineage背景铺垫：从机械控制，到软件控制
开场先抛结论：今天要讲的不是一个 shell 工具，而是代码层第一次开始出现高层闭环。Watt 和 Kubernetes 只是帮观众快速找到这条谱系。
Bridge

just-bash：把高层闭环落成工程接口

Harness Engineering 讲的是角色变化：工程师从手写实现，转向设计环境、反馈和约束。just-bash 回答的是另一个问题：这层接口具体长什么样。

Harness Engineering：测试、约束、环境、反馈一起组成高层控制回路，人类从手写实现转向设计标准与闭环。
just-bash：把文件系统、bash、自定义命令、AST 插桩拼成一个 agent 可操作、又可治理的工程接口层。

Transition

为什么要用一个长链路案例

如果只是做单一 SQL 查询，或者只跑一个分类器，我们能看到的是“模型会不会调用工具”。但要把 just-bash 讲清楚，需要一条同时包含 读文件、写代码、运行命令、生成 artifact、处理不可信文本 的完整工程链路。

所以这里选舆情应急处置：它不是为了展示算法本身，而是因为它刚好能把文件系统、自定义命令、AST 插桩、报告交付和安全边界一次串起来。

为什么它能安全改代码：OverlayFS

const bash = new Bash({ fs: overlay, cwd: overlay.getMountPoint() });

await bash.exec("cat package.json");       // 从真实磁盘读

适合代码功能场景

agent 能像在真实 repo 里工作：读源码、跑测试、生成 patch。

默认不污染宿主机

Custom Commands

为什么它不只会跑通用命令：自定义命令

defineCommand
const incidentAnalyze = defineCommand("incident-analyze", async (args, ctx) => {
  const data = await ctx.fs.readFile(args[0]);
  return { stdout: analyze(data), stderr: "", exitCode: 0 };
});

const incidentGuard = defineCommand("incident-guard", async (args, ctx) => {
  const report = await ctx.fs.readFile(args[0]);
  return checkPolicy(report);
});

不要只给模型“万能 shell”。更好的方式是把业务动作做成 可管控、可测试、可组合 的命令。

AST Instrumentation

为什么这条链路可审计：AST 与可观测性

transform pipeline
const pipeline = new BashTransformPipeline()
  .use(new TeePlugin({ outputDir: "/tmp/logs" }))
  .use(new CommandCollectorPlugin());

const result = pipeline.transform("grep -R TODO src | head");

result.script;             // 改写后的 bash
result.metadata.commands;  // ["grep", "head"]

从 AST 提取命令列表，先判断会不会触碰网络、危险命令或大范围文件。

自动插桩

用 TeePlugin 捕获每段命令输出，把 shell 过程变成可回放证据。

元数据反馈

让 harness 不只看最终 stdout，也能看“模型用了哪些动作”。

Threat Model

出界时能被拦住，才叫安全边界

判断标准：真正可用的 agent 不是“能做很多事”，而是“出界时能被稳稳拦住”。

不可信脚本

Scenario模型会生成 bash、编辑代码、运行命令，这本身就是可执行能力。
Harness用虚拟文件系统、自定义命令和执行上限把能力关进边界里。

数据注入

Scenario帖子文本可能诱导读取 host 文件、泄露信息或绕过系统策略。
Harness数据只进入分析回路，不进入控制面；host 文件与网络默认不可用。

输出污染

Scenario报告可能逐字扩散恶意文案、敏感内容或未经验证的判断。
Harness用 incident-guard 检查结构、证据引用和文本策略，再决定是否交付。

资源滥用

Scenario无限循环、大输出、深递归会让 agent 变慢、失控或拖垮环境。
Harness对命令数量、循环长度、字符串规模和运行时长做硬限制。

Production Checklist

从 demo 到生产，还要补哪些层

数据入口

把抓取、脱敏、去重、采样做成独立受控 pipeline。

分析核心

报告治理

权限分层

人工升级

审计轨迹

保存 tool call、输出、artifact diff 和最终报告。
