现场演示 · Vercel AI SDK + just-bash

舆情应急处置 Agent：如何让模型完成复杂判断链？

这个 demo 不再只是做情绪分类，而是让模型在 just-bash 工程运行时里读取多源数据、实现归一化与升级决策代码、跑测试、生成双 artifact，并判断 Nova X1 是否应该升级成 P1。重点不只是“沙箱”，还包括虚拟文件系统、自定义命令、可组合 Unix 工具，以及可用 AST 插桩扩展出的可观测性。

核心闭环

模型决定命令 → AI SDK 校验工具输入 → just-bash 执行 grep / jq / npm test / incident-analyze → stdout、stderr、exitCode、artifact 和命令元数据反馈给 harness。

演示任务

返回 Presentation
功能实现模型真实修改 `src/normalize.ts`、`src/risk-score.ts`、`src/escalation.ts`，完成从信号归一化到 P1 判断的整条链路。
反馈闭环先跑失败测试，再根据 stdout/stderr 修复，再跑 test/lint，确认归一化、聚合和决策都闭合。
模型接入真实模型模式支持 OpenAI 直连，也支持 Vercel AI Gateway：`MODEL_PROVIDER=gateway`。
工程工作台grep、jq、重定向、自定义命令和虚拟文件系统组合成完整开发环境，适合处理社媒、CSV、YAML 和 JSON 多源输入。
OverlayFS 方向生产型代码场景可以读真实 repo、写入 overlay 内存层，先 review 再落盘。
AST / 元数据just-bash 可以 parse/transform bash，用 CommandCollector 或 TeePlugin 做执行前审计和输出捕获。
可交付 artifact最终生成 `reports/escalation-decision.json` 和 `reports/incident-brief.md`，并由 `incident-guard` 检查。
安全边界帖子与工单文本都不可信；host 文件、网络、无限循环都被 harness 约束。

为什么这个 case 更适合

舆情应急处置能展示更长的工程判断链

它不是单纯分类，也不是只跑脚本。Agent 必须理解升级规则，读取社媒、工单、YAML 和产品目录，完成代码实现、风险聚合、P1 决策、artifact 交付，同时处理不可信文本注入和 shell 安全边界。

Feature Work真实修改三段业务源码，形成可 review 的实现。
Data Work读取 JSONL、CSV、YAML、JSON 与 incident log，建立完整上下文。
Decision不只分析情绪，还要判断是否升级成 P1。
Security用户文本不可信，网络和 host 文件默认不可用。

Presentation 主线

从“代码生成器”到“受控工程工作台”

这个 case 的重点不是模型会写一个关键词分类器，而是 just-bash 如何把写代码、跑测试、处理多源数据、依据规则做升级判断、生成双 artifact、命令扩展、文件系统策略和安全边界收进同一个可观察的 harness。

ObserveREADME、需求、策略、日志、社媒、工单、规则、目录数据。
Act编辑源码、运行测试、生成 JSON + Markdown 双 artifact。
Extendincident-analyze、incident-guard 这类自定义命令把业务能力变成 shell 动词。
InstrumentAST transform 可收集命令列表、插入 tee、生成审计元数据。
