just-bash threat model · 01
WHY IT EXISTS

Agent 需要 shell，但不能直接给真实 bash

核心问题不是“bash 强不强”，而是 action space 是否可控。

为什么给 bash

模型已经熟悉文件、grep、管道、重定向、脚本组合；bash 是低摩擦的通用动作语言。

为什么不能给真 bash

真实 shell 可以读 host 文件、跑二进制、访问网络、消耗资源、泄露 secrets。

just-bash 的折中

保留 shell 交互形态，但把文件系统、命令、网络、运行时都放进可审计边界。

just-bash threat model · 03
THREAT ACTORS

首要 adversary 是脚本作者

把 AI agent 当成会提交任意 bash 的不可信用户。

1A · Untrusted Script Author
  • Who: AI agent 或用户
  • Capability: 完全控制脚本输入
  • Goal: escape、读 host FS、泄露 secrets、DoS、提权
  • Trust: ZERO
1B · Malicious Data Source
  • Who: HTTP 响应、文件内容、stdin
  • Capability: 控制 expansion、变量、参数里的数据
  • Goal: prototype pollution、IFS 注入、path traversal
1C · Compromised Dependency
  • Who: npm supply-chain attacker
  • Capability: import-time code execution
  • Status: 运行时防御范围外，但属于部署加固问题
just-bash threat model · 04
TRUST BOUNDARIES

五条边界决定了防御设计

每条边界都有一个对应的失败模式。

TB1Script → Parser任意输入不能 crash、hang、泄露信息
TB2Interpreter → Filesystem必须限制在 sandbox root，阻断 symlink escape
TB3Interpreter → Network默认关闭；开启后必须 URL / method allow-list
TB4Interpreter → Host Process不得访问 env、Node internals、child_process
TB5Data → JS key space用户数据进入 object key 时防 prototype pollution

just-bash threat model · 06
DEFENSE LAYERS

Primary defense 是架构；secondary defense 是兜底

不要把 monkey-patching 误读成唯一安全边界。

Architecture没有 child_process 代码路径；命令只跑注册的 JS 实现
Virtual FSInMemoryFs / OverlayFs；symlink 默认 deny；写入可停留内存
Network默认没有 curl；开启后 URL prefix + method allow-list + redirect check
Execution Limitsloop / command / call depth / string / glob / FD 上限
Data GuardsMap、null-prototype objects、safeGet/safeSet
Defense-in-DepthFunction、eval、process.*、Module._resolveFilename 等二级阻断

just-bash threat model · 08
KNOWN GAPS

残余风险被明确写出来，反而是成熟信号

好的 threat model 不承诺绝对安全；它标注 tradeoffs 和部署条件。

Node version edge

dynamic import() 依赖 Node.js 20.6+ 的 ESM loader hook；更旧版本存在残余风险。

Secondary layer limits

pre-captured references、globalThis reassignment 说明 monkey-patching 不是 primary boundary。

Opt-in Python

Python 通过 CPython WASM 隔离，但开启后风险评级为 MEDIUM，仍需 timeout 和边界。

Output contexts

如果把 sandbox 输出渲染到网页，还要考虑 XSS / CSP。

Fuzzing scope

trap、job control、复杂 heredoc、Unicode edge cases 仍是后续加固点。

just-bash threat model · 09
OPERATING GUIDANCE

如果你要把 just-bash 接进 agent，按最小权限启用

默认安全姿态比功能完整性更重要。

1
先用 InMemoryFs 或 OverlayFs让写入可丢弃；需要真实写入时只指向 workspace sandbox。
2
网络默认关闭只给必要 URL prefix 和 HTTP methods；不要轻易 full internet。
3
Python / JS opt-in只有任务需要时打开；配置 timeout、内存和输出处理。
4
customCommands 当 trusted host code自定义命令、fetch、transform plugin 可以绕过 sandbox，必须像生产代码一样审。
5
需要任意二进制时换 VMREADME 明确建议：需要 full VM / arbitrary binary execution 时用 Vercel Sandbox。
just-bash threat model · 10
TAKEAWAY

just-bash 不是一个 VM，
而是一个可校准的 shell harness。

它把 agent 熟悉的 bash/filesystem 接口，包进一组明确的边界：虚拟文件系统、命令注册、网络 allow-list、执行上限、数据结构防护和二级逃逸阻断。

Useful

熟悉的 bash action space

Bounded

默认关闭危险能力

Observable

stdout / stderr / exit code
